Capitulo 1: Motivación

Control de versiones. Definición y objetivos.

Sin control de versiones.

  • Copiar ficheros y mandar correos no escala.
  • Sólo tenemos la versión actual.
  • Colaborar es dificil. ¿Quién tiene la última versión?

Control de versiones. Definición y objetivos.

Con control de versiones

  • Se mantiene un historial de cambios y colaboraciones.
  • Se puede colaborar de manera ordenada con otros. Manteniendo un registro de cambios.
  • Permite realizar metricas acerca de como crece un proyecto y colabora un equipo.
  • Ayuda a mejorar la gestión de cambios y los aspectos de configuración del software.
  • Ayuda a mejorar la calidad del software.

Historia de Git

  • Git surge en el entorno del desarrollo de Linux.
  • antes de 02/2002. Para desarrollar Linux, se usaban parches y tar.gz (y sin UML ni tonterias de esas.)
  • 02/2002.Bitmover regala la licencia de BitKeeper un SCM distribuido privativo.
  • 04/2005. BitMover retira la licencia tras roces.
  • 04/2005. Linus Torvalds presenta Git, que ya reúne ramas.
  • 06/2005. Git se usa para gestionar Linux
  • 02/2007. Git 1.5.0 es utilizable por mortales
  • 11/2011. Git 1.7.7.3
